◆Git開発入門


用語解説


リポジトリ

プロジェクトファイルや、更新履歴などGitで共有するすべての情報をまとめる倉庫。
開発時には、ネットワーク上にある基準となるリモートリポジトリと開発者自身のローカル環境に構築するローカルリポジトリがあり 開発者はローカルリポジトリを編集して開発を進めます。
ローカルで作業した内容をリモートリポジトリ上にアップロードすることで他の開発者と編集済みの情報を共有することができる



クローン

リモートリポジトリを自身のローカル環境にすべてコピーしローカル環境を構築することを指します。
これをすることで、リモート上にある本体を触ることなく自身の環境で開発が進めることができチームでの並行作業を実現できます。

Gitコマンド
1.|$ git clone <URL> //ローカルリポジトリを作成


ブランチ

作業をする際、ブランチを作成することで現時点の作業点から分岐し本体（メインブランチ）に影響を与えずにファイルの編集ができる。 そのため、ほかの人が同じファイルを編集しようとしても影響されることがなく、並行作業ができたり一時的なテストを行いたいときなどにも簡単に行えるため、作業を始める際は先にブランチを切ってからスタートするとよいです。

チェックアウト

ブランチを移動するときに使うコマンド。ブランチを切ったタイミングでは移動しないので必ず行う。
<-b> のオプションをつけることでブランチを作成して移動することもできる

Gitコマンド
1.|$ git branch <ブランチ名> //ブランチの作成
2.|$ git checkout <ブランチ名> //ブランチの移動
3.|$ git checkout -b <ブランチ名> //ブランチの移動と作成


コミット

ブランチ内での作業を確定させ、編集履歴を残すことを指します。コミットをする際はどの部分を変更したかの編集差分の確認もでき、 自身で何をしたのかが簡単にわかるようになっています

ワークツリー

編集している一時作業領域のこと。ここでチェックアウトをしてしまうと情報が消えてしまうため注意が必要。ほかのブランチの情報を知りたい場合はワークツリーコマンドを実行することで編集内容を失うことなく確認することができる

ステージ

ファイルに変更を加えるとワークツリーからステージに移行する。ステージの状態に入ったファイルをコミットすることで編集を加えたファイルのみをコミットすることができる。

Gitコマンド
1.|$ git commit <コミット名> //作業をコミットする
2.|$ git add //作業をステージに上げる
3.|$ git worktree <ブランチ名> //指定したブランチのワークツリーを確認する


マージ

メインブランチと作成したブランチを統合し、編集履歴を反映させることをマージといいます。
マージをする際、ほかのコミットと競合しうまく統合できない場合があるため、慎重に作業が必要です。
マージと似た操作でリベースというものがあります。

リベース

複数あるコミットを一つにまとめる時、元のブランチから分岐したブランチを統合するときなどに使う。
マージとの違いは編集履歴をそのまま元のブランチの後ろに付ける。

マージの手順

  1. マージ先のブランチに移動
    		2.

    本体となるブランチに移動する。
  2. マージを実行
    		3.

    統合したい内容が格納されているブランチを指定して実行
  3. 確認
Gitコマンド
1.|$ git merge <マージ先のブランチ名> //他のブランチの内容を現在のブランチに適用する
2.|$ git rebase <マージ先のブランチ名> //他のブランチの内容を現在のブランチにリベースする


プル

プルはリモートリポジトリからすべての更新情報を自身のローカルリポジトリ環境に適用することを指します。
これをすることにより、最新の状態から作業を始めることができます。そのため、作業を始める前にプルをしておくことが大事です。 また、自身の作業が終わりリモートリポジトリにアップロードした際にもしておくとよいです。

フェッチ

プルを行う際、先に実行すると現時点のリモートリポジトリの状態を取得できる。
内容を確認し競合がありそうなところを手動で直してからプルをすることで安全に情報を更新できる。

Gitコマンド
1.|$ git pull //リモートリポジトリの内容をローカルリポジトリに統合
2.|$ git fetch //リモートリポジトリの更新情報を取得


プッシュ

プッシュはプルと逆でローカルリポジトリでの更新情報をリモートリポジトリに適用することができます。Git開発の中では一番リスクの高い処理のため慎重に行います。

Gitコマンド
1.|$ git push //ローカルリポジトリの内容をリモートリポジトリに統合
