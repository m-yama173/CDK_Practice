◆Git開発入門

開発の流れ

  1. リモートリポジトリを作成(ホスト側)
    		2.

    GitHubなどを利用してリモートリポジトリを作成
    ※以下、GitHubのプライベートリポジトリを例に説明。
  2. 共同で作業するための準備(ホスト側)
    		3.

    チームメンバーをプライベートリポジトリ招待する
  3. チーム招待を承認(参加メンバー側)
    		4.

    参加メンバーはGitHubアカウントにログインし、招待を承認
  4. Cloneを使いローカルリポジトリを作成
    		5.

    ローカルリポジトリを作成し、ローカル環境を構築
  5. コードエディタでプロジェクトを開く (VsCode推奨)
    		6.

    プロジェクトがローカル環境にあるかの確認
  6. 開発開始！

開発進行方法

  1. ブランチを作成
    		2.

    ブランチを作成し作業場所を切り離す
  2. 変更の反映
    		3.

    コミットすると現在のローカルブランチに変更記録が残る
  3. マージ
    		4.

    マージをすることで切り離したブランチの内容を本体に反映
  4. フェッチ

    5. 現時点でサーバー側に変更がないかの確認をする

  5. プル
    		6.

    変更を加えられたリモートリポジトリの内容をローカルへ反映
    サーバーに更新があれば、ローカルの内容とマージする
  6. プッシュ
    		7.

    ローカルリポジトリからリモートリポジトリに反映させ全体に共有
  7. 最初に戻る

注意点

プッシュするとき、フェッチ、プルを先に行わないとデータを壊してしまう可能性があります。
先にフェッチをすることでリモートリポジトリの更新履歴を取得することができ、変更されているかどうかの確認ができます。
その後、プルをすることでローカルリポジトリとリモートリポジトリの差異を確認しながらマージ作業を行うことができ、安全に更新ができます。
